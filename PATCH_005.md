## 1. 認証システム実装

### 1.1 概要

2段階の認証方式を実装する。

| モード | 認証方式 | 対象者 | 用途 |
|--------|---------|--------|------|
| スタッフモード | 4桁PIN（共通） | 全スタッフ | 日常の予約操作 |
| 管理者モード | ID + パスワード | 院長のみ | 設定変更・マスタ管理 |

### 1.2 運用イメージ

```
朝、医院PCでブラウザを開く
  → PIN入力画面（4桁テンキー風UI）
  → 4桁入力 → スタッフモードでログイン
  → 1日中開きっぱなし（セッション持続）
  → 閉じるまでログアウトしない

設定変更が必要なとき（院長のみ）:
  → ヘッダーの[設定]メニュー → 管理者ログインを求められる
  → ID + パスワード入力 → 管理者モードに昇格
  → 設定変更完了後 → 自動でスタッフモードに戻る（or 手動ログアウト）
```

### 1.4 バックエンド実装

#### DB: settings テーブルに追加

```python
# 初期データ（seed.py に追加）
{"key": "staff_pin", "value": "1234"},           # スタッフPIN（初期値、管理画面で変更可）
{"key": "admin_username", "value": "admin"},      # 管理者ID（初期値）
{"key": "admin_password_hash", "value": "..."},   # bcryptハッシュ
```

※ 管理者パスワードはハッシュ化して保存。平文で保存しない。

#### 認証API

| Method | Path | 説明 |
|--------|------|------|
| POST | /api/auth/staff-login | PIN認証 → スタッフトークン発行 |
| POST | /api/auth/admin-login | ID+パスワード認証 → 管理者トークン発行 |
| POST | /api/auth/logout | ログアウト |
| GET | /api/auth/me | 現在のログイン状態取得 |
| PUT | /api/auth/change-pin | PIN変更（管理者のみ） |
| PUT | /api/auth/change-password | 管理者パスワード変更（管理者のみ） |

#### トークン方式

- JWT（JSON Web Token）を使用
- スタッフトークン: `role: "staff"`, 有効期限24時間（朝入れたら翌朝まで持つ）
- 管理者トークン: `role: "admin"`, 有効期限30分（設定変更後は自動的に期限切れ）
- トークンはlocalStorageに保存
- APIリクエスト時にAuthorizationヘッダーに付与

#### ミドルウェア

```python
# 全APIに認証チェック
# - /api/auth/* → 認証不要
# - /api/chatbot/* → 認証不要（患者向けチャットボット）
# - /api/settings/*, /api/practitioners POST/PUT/DELETE,
#   /api/menus POST/PUT/DELETE, /api/reservation-colors POST/PUT/DELETE
#   → 管理者トークン必須
# - それ以外 → スタッフトークン以上
```
