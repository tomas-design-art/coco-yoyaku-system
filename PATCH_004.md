# 固定色修正指示（PATCH_004）

**Date:** 2026-03-22
**目的:** 警告色・ステータス色の視認性改善。オレンジ系が多すぎて見分けがつかない問題を解消する。

---

## 変更前（現状）

| ステータス | 色 | 問題 |
|-----------|-----|------|
| 競合エラー | #EF4444（赤） | OK |
| キャンセル申請 | #FCA5A5（薄赤） | 競合エラーと紛らわしい |
| 変更申請 | #FB923C（オレンジ） | 初診（ユーザー設定オレンジ）と紛らわしい |
| 仮確保(HOLD) | #8B5CF6（紫） | OK |
| 仮予約(PENDING) | #F59E0B（黄寄りオレンジ） | 変更申請と紛らわしい |

## 変更後

| ステータス | 新しい色 | HEX | 意図 |
|-----------|---------|------|------|
| 競合エラー | **濃い赤** | #DC2626 | 一番目立つ。見た瞬間「ヤバい」とわかる |
| キャンセル申請 | **グレー（取り消し線風）** | #9CA3AF | デッド感。「もう消える予約」とわかる |
| 変更申請 | **濃い黄色** | #EAB308 | 「対応が必要」の警告色。オレンジとは明確に違う |
| 仮確保(HOLD) | **紫** | #8B5CF6 | 変更なし。「一時的な特殊状態」として独立 |
| 仮予約(PENDING) | **濃い黄色** | #EAB308 | 変更申請と同じ黄色。「まだ確定してない＝対応必要」で統一 |

### 視覚的な整理

```
■ #DC2626  競合エラー     → 赤「危険！すぐ対応」
■ #9CA3AF  キャンセル申請  → グレー「消える予約」
■ #EAB308  変更申請       → 黄「対応が必要」
■ #EAB308  仮予約         → 黄「対応が必要」（変更申請と同系色で統一）
■ #8B5CF6  仮確保(HOLD)   → 紫「一時キープ中」
```

### ユーザー設定色との棲み分け

初期設定色（管理者が変更可能）:
- 保険診療: #3B82F6（ブルー）
- 自費診療: #10B981（グリーン）
- 初診: #F97316（オレンジ）

固定色はすべて**赤・グレー・黄・紫**で、ユーザー設定色の**青・緑・オレンジ**と完全にカテゴリが分かれる。これで混同しない。

### キャンセル申請ブロックの追加表現

グレーだけだと「空き」に見える可能性があるので、以下も併用する:

```typescript
// キャンセル申請の予約ブロック
{
  backgroundColor: '#9CA3AF',  // グレー
  opacity: 0.7,                // やや透過
  textDecoration: 'line-through', // 取り消し線
  borderStyle: 'dashed',       // 破線ボーダー
}
```

これで「キャンセル予定だけどまだ枠は押さえてる」感が出る。
