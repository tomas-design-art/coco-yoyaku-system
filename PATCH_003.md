# UI改善指示書（PATCH_003）— 施術者表示とタイムテーブルレイアウト

**Date:** 2026-03-22
**対象:** 現在のタイムテーブルUI（添付スクリーンショット参照）

---

## 1. 施術者の登録と表示/非表示

### 1.1 登録上限
- 施術者は**上限なし**で登録可能（実運用で10人程度を想定）
- 既にそうなっていれば変更不要

### 1.2 表示/非表示設定

施術者管理画面（/settings/practitioners）に**「予約画面に表示」トグル**を追加する。
既存の `is_active` とは別の概念。

| フィールド | 意味 | 用途 |
|-----------|------|------|
| `is_active` | 在籍中か（論理削除用） | false = 退職者。システムから完全に隠す |
| `is_visible`（新規） | 予約画面に表示するか | false = 休職中・一時非表示。管理画面には残る |

**DBマイグレーション:**
```sql
ALTER TABLE practitioners ADD COLUMN is_visible BOOLEAN DEFAULT TRUE;
```

**影響:**
- タイムテーブルの施術者列: `is_active=true AND is_visible=true` のみ表示
- 施術者管理画面: `is_active=true` をすべて表示し、`is_visible` トグルを操作可能
- 予約登録モーダルの施術者選択: `is_active=true AND is_visible=true` のみ

## 2. タイムテーブルレイアウトの改善

### 2.1 デフォルト表示

- **デフォルトは「週」表示**（現状通り）
- 「日」「週」ボタンは現状の位置でOK

### 2.2 週表示での施術者並列表示

**週表示では、各曜日の列の中に施術者サブ列を並べて表示する。**

現在のスクリーンショットは「1施術者の1週間」を表示しているが、改善後は「全施術者の1週間」をデフォルトで表示する。

#### レイアウトイメージ（施術者3人の場合）

```
┌──────────────────────────────────────────────────────────────────────┐
│  [←] 3/16 〜 3/22 [→] [今日]    [日][週]  [院長][施術者A][施術者B]  │
├──────┬──────────────────────┬──────────────────────┬─────────────────┤
│      │     3/16(月)         │     3/17(火)         │    ...          │
│ 時間 ├──────┬──────┬────────┼──────┬──────┬────────┤                 │
│      │ 院長 │ A    │ B      │ 院長 │ A    │ B      │                 │
├──────┼──────┼──────┼────────┼──────┼──────┼────────┤                 │
│09:00 │ ██   │      │        │      │ ██   │        │                 │
│09:15 │ ██   │      │        │      │ ██   │        │                 │
│09:30 │ ██   │      │        │      │ ██   │        │                 │
│...   │      │      │        │      │      │        │                 │
```

#### レスポンシブ自動判断ロジック

```typescript
// 施術者の並列表示数を画面幅から自動計算

function getVisiblePractitionerCount(
  screenWidth: number,
  totalPractitioners: number,
  viewMode: 'day' | 'week'
): number {
  const timeColumnWidth = 50;        // 時間列の幅(px)
  const availableWidth = screenWidth - timeColumnWidth;

  if (viewMode === 'day') {
    // 日表示: 1日しかないので全員表示可能
    // 最小列幅120pxを確保
    const minColumnWidth = 120;
    return Math.min(totalPractitioners, Math.floor(availableWidth / minColumnWidth));
  }

  // 週表示: 7日 × N人
  const daysToShow = 7;
  const minSubColumnWidth = 80;      // 施術者サブ列の最小幅(px)
  const minDayColumnWidth = 60;      // 曜日列の最小幅（施術者1人分）

  // 全員を表示した場合の1サブ列幅を計算
  const subColumnWidth = availableWidth / (daysToShow * totalPractitioners);

  if (subColumnWidth >= minSubColumnWidth) {
    // 全員表示しても読める → 全員表示
    return totalPractitioners;
  }

  // 全員は厳しい → 画面に収まる最大人数を計算
  const maxPractitioners = Math.floor(availableWidth / (daysToShow * minSubColumnWidth));
  return Math.max(1, maxPractitioners);
}
```

#### 表示パターンまとめ

| 施術者数 | 画面幅 | 週表示の動作 |
|---------|--------|-------------|
| 1人 | どの幅でも | 1人の列が全幅を使用。見やすい |
| 2人 | どの幅でも | 2人並列。各曜日内に2サブ列 |
| 3人 | 1280px以上 | 3人並列。各曜日内に3サブ列 |
| 3人 | 1280px未満 | 自動で表示人数を減らす。施術者ボタンでON/OFF |
| 4人以上 | 1920px | 可能なら全員、無理なら画面幅に応じて削減 |
| 4人以上 | 1280px | 2〜3人まで表示。残りはボタンで切替 |

### 2.3 日表示での施術者並列表示

日表示は1日分だけなので、施術者全員を横並びで表示できる。
現状のスクリーンショットの「施術者プルダウン」は日表示では不要になる（全員表示するため）。

ただし施術者が多い場合（6人以上等）は横スクロールが必要になるので、
日表示でも施術者ボタンでのON/OFFは有効にしておく。

## 3. 施術者選択UI（プルダウン → ボタン）

### 3.1 現状の問題

右上のプルダウン（「田中 太郎 ▼」）は：
- 1クリックでプルダウンを開く → 2クリック目で選択 → 計2クリック
- 現在誰が表示されているかが一目でわからない
- 複数人同時表示の制御ができない

### 3.2 改善後のUI

プルダウンを**トグルボタン群**に変更する。

```
[日][週]    [院長 ✓][施術者A ✓][施術者B ✓]
```

- 各施術者名がボタンになっている
- **ON（表示中）**: ボタンが塗りつぶし（アクティブ色）
- **OFF（非表示）**: ボタンがアウトライン（グレー）
- クリックで即トグル → タイムテーブルに即反映
- **最低1人はON**（全員OFFにはできない）
- ボタンの色は施術者ごとに区別しやすいように（任意）

#### モックアップ

```
全員ON:
[院長 ✓] [施術者A ✓] [施術者B ✓]
 (blue)    (blue)       (blue)

施術者Bだけ非表示:
[院長 ✓] [施術者A ✓] [施術者B]
 (blue)    (blue)      (gray/outline)

院長だけ表示:
[院長 ✓] [施術者A] [施術者B]
 (blue)   (gray)    (gray)
```

#### 動作仕様

| 操作 | 結果 |
|------|------|
| ONのボタンをクリック | OFFにする（他に1人以上ONなら） |
| OFFのボタンをクリック | ONにする |
| 最後の1人をOFFにしようとする | 何も起きない（or トースト「1人以上表示してください」） |

#### 初期状態
- **画面読み込み時**: レスポンシブ自動判断で全員ON or 画面幅に収まる人数をON
- **ユーザーが手動でON/OFF**: その選択を優先（sessionStorageで保持）

### 3.3 レイアウト配置

```
┌─────────────────────────────────────────────────────────────────┐
│ [←] 3/16 〜 3/22 [→] [今日]                                     │
│                                                                  │
│ [日][週]    [院長 ✓][施術者A ✓][施術者B ✓]    🔇 🔔3            │
├──────────────────────────────────────────────────────────────────┤
```

- 左: 日/週ボタン + 施術者ボタン群
- 右: 音量・通知
- 施術者ボタンが多い場合（5人以上）は横スクロール or 2行に折り返し

## 4. 施術者1〜2人の場合の幅調整

施術者が少ない場合、列幅が広すぎて間延びしないようにする。

### ルール

```typescript
function getDayColumnStyle(practitionerCount: number, viewMode: 'day' | 'week') {
  if (viewMode === 'week') {
    // 週表示: 7日で画面を分割。各日の中でサブ列を等分
    // 施術者1人でも7等分は変わらない → 1人なら1日分の幅全体を使う
    return { flex: '1 1 0%' };  // 均等分割
  }

  if (viewMode === 'day') {
    // 日表示: 施術者列を等分
    // ただし、1人の場合に画面全幅は広すぎるので max-width を設定
    const maxWidth = practitionerCount === 1 ? '600px'
                   : practitionerCount === 2 ? '400px'
                   : undefined;  // 3人以上は均等分割でOK
    return { flex: '1 1 0%', maxWidth };
  }
}
```

### 表示イメージ

**施術者1人・週表示:**
```
│  月  │  火  │  水  │  木  │  金  │  土  │  日  │
│ 院長 │ 院長 │ 院長 │ 院長 │ 院長 │ 院長 │ 院長 │
→ 各曜日の全幅を1人で使用。予約ブロックが大きく見やすい
```

**施術者2人・週表示:**
```
│    月     │    火     │    水     │ ...
│ 院長│ A  │ 院長│ A  │ 院長│ A  │
→ 各曜日を2分割。まだ十分読める
```

**施術者3人・週表示:**
```
│     月      │     火      │ ...
│ 院長│ A │ B │ 院長│ A │ B │
→ 各曜日を3分割。フルHD(1920px)なら問題なし
```
